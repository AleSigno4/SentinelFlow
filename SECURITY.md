# Security Policy

This is a personal portfolio project, not production software — please keep that in mind when reporting issues.

## Reporting a Vulnerability

If you find a security issue (e.g. authentication bypass, injection, exposed secrets), please **do not open a public issue**. Instead, reach out directly via [tua email o GitHub profile].

## Known limitations (by design, for a demo project)

- Default demo credentials are documented in the README and are not meant for real deployment.
- The H2 file-based database is for local demo purposes only.
- No rate limiting / brute-force protection is currently implemented.
